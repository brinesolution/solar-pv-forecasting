\section{Experimental Setup}
\label{sec:experimental}

The experimental protocol uses fixed chronological splits and a consistent preprocessing and model-evaluation configuration. Training uses the earliest chronological block, validation begins on 2024-01-01, and the untouched test interval begins on 2024-12-28. No random row shuffle is used. Metrics and figures are computed from the cleaned dataset using the same feature-generation, model-inference, Stage-1 clipping, and post-processing definitions described in this paper. The test should be read as an offline target-weather-input evaluation: it does not include archived issue-time forecasts, forecast lead times, or weather-source forecast error.

\subsection{Metrics}

For observations $y_i$ and predictions $\hat{y}_i$, the mean absolute error is
\begin{equation}
\mathrm{MAE}=\frac{1}{N}\sum_{i=1}^{N}|y_i-\hat{y}_i|.
\label{eq:mae}
\end{equation}
The root mean square error is
\begin{equation}
\mathrm{RMSE}=\sqrt{\frac{1}{N}\sum_{i=1}^{N}(y_i-\hat{y}_i)^2}.
\label{eq:rmse}
\end{equation}
The normalized MAE and RMSE are reported relative to plant rating:
\begin{equation}
\mathrm{nMAE}=\frac{\mathrm{MAE}}{\Prated}\times100\%, \qquad
\mathrm{nRMSE}=\frac{\mathrm{RMSE}}{\Prated}\times100\%.
\label{eq:normalised_errors}
\end{equation}
The coefficient of determination is
\begin{equation}
R^2=1-\frac{\sum_{i=1}^{N}(y_i-\hat{y}_i)^2}
{\sum_{i=1}^{N}(y_i-\bar{y})^2}.
\label{eq:r2}
\end{equation}
The Nash-Sutcliffe efficiency (NSE) is computed with the same residual-to-variance ratio:
\begin{equation}
\mathrm{NSE}=1-\frac{\sum_{i=1}^{N}(y_i-\hat{y}_i)^2}
{\sum_{i=1}^{N}(y_i-\bar{y})^2}.
\label{eq:nse}
\end{equation}
Forecast skill against a baseline $b$ is
\begin{equation}
\mathrm{Skill}_b=1-\frac{\mathrm{RMSE}_{\mathrm{model}}}
{\mathrm{RMSE}_{b}}.
\label{eq:skill}
\end{equation}
Skill is reported only when the model and baseline are evaluated on the same rows.

\subsection{Persistence Baselines}

Two persistence baselines are reported because they answer different operational questions. The 5-min last-value baseline is
\begin{equation}
\hat{y}^{\mathrm{LV}}(t)=y(t-5~\mathrm{min}).
\label{eq:last_value_persistence}
\end{equation}
This is a strong nowcast because it uses live plant output immediately before the target timestamp. The previous-day same-time baseline is
\begin{equation}
\hat{y}^{\mathrm{PD}}(t)=y(t-24~\mathrm{h}).
\label{eq:previous_day_persistence}
\end{equation}
This is a simpler schedule-oriented forecast baseline. The proposed model is compared against both on common daylight masks. Reporting both prevents an inflated skill claim: the proposed weather-input path improves over previous-day persistence, but it does not beat the 5-min last-value nowcast under the current test protocol.

\subsection{Ablations}

Two ablations are evaluated with the same feature pipeline. The first is a single-stage XGBoost model that predicts power directly from weather, time, solar, and cloud features without the learned $\kt$ intermediate. The second preserves the two-stage cascade but removes low-, mid-, and high-cloud channels, leaving only total cloud cover and its derived lag/rolling features. Both ablations reuse the chronological split and tuned parameter family so the comparison isolates model structure rather than data leakage or a different test interval.

\subsection{Implementation Notes}

The implementation supports GPU-accelerated XGBoost for faster experimentation, with CPU execution as an equivalent fallback. Hardware acceleration is not a scientific contribution of the paper; it affects only training convenience. The deployment inference path accepts a weather forecast CSV, interpolates and engineers features, loads the saved models, clips the prediction envelope, and writes prediction outputs. Fig.~\ref{fig:inference} summarizes this path; the reported test metrics still come from the offline held-out target-weather-input protocol.
